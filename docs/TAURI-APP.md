# AGENTS.md — tauri-app

> Desktop UI for story visualization
> Last updated: 2026-04-03

---

## Role

Desktop application for visualizing and interacting with story graphs.
Built with React + Vite + Tauri.

---

## Input

- **Primary**: StoryGraph JSON (from MCP server output or file upload)
- **Secondary**: User edits and interactions
- **Tertiary**: Export format selections (Mermaid, Canvas, Markdown, Dashboard)

---

## Output

- **UI Components**:
  - StoryGraph visualization (interactive diagram)
  - Character cards (details, relationships, arcs)
  - Event timeline (3-act structure)
  - Conflict tracker
  - Validation results display
- **Exported Files**:
  - Mermaid diagrams
  - Canvas JSON
  - Markdown documents
  - HTML dashboards

---

## Dependencies

### Required
- `@bl1nk/visual-mcp` — StoryGraph types, export functions
- `@tauri-apps/api` — Tauri core API
- `@tauri-apps/plugin-shell` — Shell access
- `react`, `react-dom` — UI framework

### Development
- `@tauri-apps/cli` — Tauri CLI
- `vite` — Build tool
- `typescript` — Type checking
- `vitest` — Testing

---

## Data Flow

```
1. User opens app
   ↓
2. Load StoryGraph JSON (file or MCP call)
   ↓
3. Parse and validate StoryGraph
   ↓
4. Render UI components
   ↓
5. User interacts (edit, export, validate)
   ↓
6. Export to selected format
   ↓
7. Save or share output
```

---

## Integration with @bl1nk/visual-mcp

The Tauri app consumes the MCP server output in two ways:

1. **Direct Import**: Import types and functions from `@bl1nk/visual-mcp`
   ```typescript
   import type { StoryGraph } from '@bl1nk/visual-mcp';
   import { validateGraph, toMermaid } from '@bl1nk/visual-mcp';
   ```

2. **File-based**: Read StoryGraph JSON files generated by MCP tools
   ```typescript
   const graph = JSON.parse(await readTextFile('story.json'));
   ```

## Architecture Notes

- UI is built with React + TailwindCSS
- State management via React hooks (no external state library)
- File I/O via Tauri's fs API
- No direct MCP server connection (file-based or import-based)
