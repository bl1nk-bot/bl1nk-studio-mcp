# 📝 โปรโตคอลการจัดการงาน (Universal Todo Protocol)

จงทำตามขั้นตอนเหล่านี้เพื่อให้มั่นใจว่าระบบ "ความจำ" ของโปรเจกต์จะไม่มีวันขาดหาย:

## 1. การตรวจสอบและสร้าง (Check or Create)
- ตรวจสอบว่ามีไฟล์ `TODO.md` ใน Root ของโปรเจกต์หรือไม่
- **หากไม่มีไฟล์:** ให้ใช้เครื่องมือ `write_file` สร้างไฟล์ใหม่ทันทีโดยใช้โครงสร้างมาตรฐานนี้:
  ```markdown
  # TODO.md — [ชื่อโปรเจกต์]
  
  > **Status:** INITIALIZING | **Last Updated:** [วันที่ปัจจุบัน]
  
  ---
  ## 🚨 EMERGENCY
  
  ## 🔴 High Priority
  
  ## 🟡 Core Development
  
  ## ✅ Recently Completed (Verified)
  ```

## 2. การวิเคราะห์งาน (Task Analysis)
- **รายละเอียดงาน:** รับมาจากอาร์กิวเมนต์
- **หมวดหมู่:** ถ้าผู้ใช้ไม่ระบุ ให้จัดลงใน `🔴 High Priority` เป็นค่าเริ่มต้น
- **อัปเดต Metadata:** ต้องเปลี่ยนวันที่ `Last Updated` ทุกครั้งที่มีการแก้ไขไฟล์

## 3. ขั้นตอนการเขียน (Strict Writing)
- **ห้ามลบ:** ห้ามเขียนทับหรือลบเนื้อหาเดิม โดยเฉพาะในหมวด `Recently Completed`
- **แทรกด้านบน:** งานใหม่ต้องอยู่ด้านบนสุดของหมวดหมู่ เพื่อให้สถาปนิกเห็นได้ทันที
- **รูปแบบ:** `- [ ] type:fix/feat รายละเอียดงาน` (ต้องเป็นภาษาไทย)

## 4. การปิดงาน (Archiving)
- เมื่อเปลี่ยนสถานะเป็น `[x]` ต้องย้ายงานนั้นจากหมวดเดิมลงไปไว้ที่ `## ✅ Recently Completed (Verified)` ทันที
- ห้ามทิ้งงานที่เสร็จแล้วไว้ในหมวด Active

## 5. การเรียนรู้ (Learning)
- รายงานผลให้สถาปนิกทราบว่า: "ตรวจพบ/ไม่พบไฟล์เดิม จึงได้ดำเนินการ [เพิ่ม/สร้างใหม่] เรียบร้อยแล้ว"
